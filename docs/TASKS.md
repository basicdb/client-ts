- **UNPUBLISHED_SCHEMA_UX:** When schema is at version 0 (unpublished), the error flow is confusing — logs say "Schema is invalid!" when the schema structure is actually fine, it just hasn't been published yet. Currently partially fixed (better log message + `status: 'unpublished'`), but should also surface a clear user-facing error/banner (e.g. "Publish your schema at basic.tech to enable sync") instead of the generic "Schema is invalid!" error state.
- **AUTH_DOCS:** Document token handling behavior in readme/docs: getToken() 5s buffer, force-refresh on 401, mutex behavior, and guidance on using the same getToken() from context for custom API calls.
- **SIGNOUT_RELOAD:** `handleSignOut` in `BasicProvider` and the cross-tab `signed_out` handler in `AuthManager` currently call `window.location.reload()` because the sync layer (Dexie + syncable) doesn't cleanly re-initialize after teardown. Replace the reload with proper sync DB teardown + re-init so sign-out → sign-in works without a full page reload, and so cross-tab sign-out can clean up the sync connection and IndexedDB without reloading the receiving tab.
- **AUTH_STATUS_ERGONOMICS:** `isSignedIn` is intentionally `true` during `authStatus === 'reauth_required'` so the UI can display user info while prompting re-auth. This can be confusing for consumers who use `isSignedIn` to toggle sign-in vs sign-out buttons. Consider adding a convenience getter like `needsReauth` or `shouldPromptSignIn` so consumers don't need to inspect `authStatus` directly. Revisit naming of the `AuthStatus` union values for clarity.
